Generic MOBA

Use Space to lock camera/free look, right click to move and Q & E do cast a kick and a roll. Hit that dummy!

A game being developed for my Xp Criativa class, which focus on a better game feel and polishing rather than a finished product. It's a good demonstration of what I can do all around Unity, as I'm developing this one alone.

As in right now, I've implemented some programming techniques, such as Object Pooling and Finite State Machine (with separate classes and interfaces rather than using case switch).

Devlog #1

The first sprint was guided towards building a game ideia for the game and presenting for the class. You can check out the small presentation (in portuguese)

Devlog #2

This sprint served as a guide to what I should do with the project, as my peers and my teachers gave me advice after watching my small pitch. My focus with the game is to give a basic 'training mode' feeling of a MOBA. I had experience learning about realtime multiplayer games, and saw this as a focus wide away from what I expect and for what I have to achieve in class.

Devlog #3

One of the objectives of this sprint was to delivery some basic screenshots (made in photoshop or some basic in game content) of what would be the main aspects trying to achieve in the class, you can have a quick look in my montage as highlighted

Devlog #4

As I had my ideia better guided and already had some research about assets and general information about the project theme. I imported some of the assets from a similar project I'm involved, Poke the Ogre, and started developing basic aspects of the game.

Devlog #5

With the base already set. This sprint was more focused in giving a first test of polishing for the game, This quick timelapse show some progress of the kick skill I got the base slash art in unity asset store and spent some time implementing it and adjusting to fit the game context and color pallete.

Devlog #6

This sprint was very code-driven, as I had some awesome insights in my AI class, I changed all the codebase to fit properly a class based State Machine rather than using old case switching. The code is made by base abstract classes and a manager script only for the states. I also implemented the skill system in a similar way.

Devlog #7

After a hard sprint, I used some of my professors feedback from the project and focused on giving a move vivid scenario. I got some assets from the asset store and used all my artistic power to create a simple, but quite convincing environment. This timelapse show me giving live to the prototype's scene.

Devlog #8

Back to hard coding, I used this week mainly to finish all the State Machine and Skill system basics, with the feedback given in class from the mid-term presentation to all class, I polished mainly the skill casting, positioning and skill queue, AND added a fullscreen button (Professor Vermonde is probably happy now).
